# Value Lab v0.6.1 逻辑层级重构计划

## 目标

不扩充新品类，不改变现有价格模型；先修复全站信息依赖顺序，使每个结果都能找到它的前置对象和条件。

## 1. 成本页

```text
步骤 1 产品与配置
→ 步骤 2 新品价格基准
→ 步骤 3 我的持有条件
→ 步骤 4 当前持有成本
→ 步骤 5 价值趋势
→ 步骤 6 交易路径与换机结论
```

实施：

- 将产品类别、机型、容量 / 内存 / SSD / 网络独立为 `costProductControls`。
- 将买入、卖出、持有、折旧、费用独立为 `costHoldingControls`。
- 新品价格移至产品选择之后。
- KPI 独立放入结果区。
- 每一块增加紧凑步骤编号与当前对象说明。

## 2. 市场页

```text
步骤 1 产品与配置
→ 步骤 2 市场筛选
→ 步骤 3 当前价格摘要
→ 步骤 4 价格记录
```

实施：

- 新增机型、容量 / 内存 / SSD / 网络选择。
- 市场记录默认只显示当前明确配置。
- 新增数据日期筛选。
- KPI 只对当前配置和当前过滤条件统计。

## 3. 购买参考页

```text
步骤 1 选择类别与需求
→ 步骤 2 选择产品
→ 步骤 3 查看产品详情
```

实施：

- 不再自动选择第一条产品。
- 只有点击行或从其他页面明确带入时显示详情。
- 过滤后若选中产品不再存在，清空详情。

## 4. 生活平替页

```text
步骤 1 搜索、分类与筛选
→ 步骤 2 选择方案
→ 步骤 3 查看详情
```

实施：

- 不再自动选择第一条。
- 过滤后选中项消失时清空详情。
- 详情区保持轻量占位，等待用户选择。

## 5. 我的页面

```text
步骤 1 添加或查看我的物品
→ 步骤 2 查看资产汇总
→ 步骤 3 查看平替收藏
```

实施：

- 无物品时隐藏资产 KPI，显示明确空状态。
- 添加弹窗新增内存、容量 / SSD、网络配置。
- 保存显式配置字段。
- 当前估值优先使用保存的具体配置。

## 6. 文档与测试

- 更新最终网站思维导图。
- 更新数据模型中的本地物品字段。
- 更新 Changelog 与版本号。
- 自动测试桌面与 390px 移动端。
- 测试前置选择、结果显隐、状态恢复、跨页面带入和旧数据兼容。
